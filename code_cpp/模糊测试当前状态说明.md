## 如何判断是否发现崩溃

### 运行模糊测试时

运行 `afl-fuzz` 后，你会看到实时统计界面，其中会显示：

```
┌─ process timing ──────────────────────────────────────┐
│        run time : 0 days, 0 hrs, 5 min, 23 sec       │
│   last new path : 0 days, 0 hrs, 0 min, 12 sec       │
│ last uniq crash : 1    ← 如果发现崩溃，这里会显示数字 │
│  last uniq hang : none seen yet                       │
└───────────────────────────────────────────────────────┘
```

- **`last uniq crash : 0`** = 未发现崩溃
- **`last uniq crash : 1`** = 发现1个崩溃
- **`last uniq crash : 2`** = 发现2个崩溃

### 检查崩溃文件

```bash
# 查看是否有崩溃文件
ls fuzz_output/default/crashes/

# 如果目录为空或不存在，说明未发现崩溃
# 如果有文件（如 id:000000*），说明发现了崩溃
```

## 运行时间要求

根据实验要求：
- **至少运行5小时**
- 如果5小时内发现崩溃，可以提前停止
- 如果5小时后仍未发现崩溃，需要提供运行时间证明
